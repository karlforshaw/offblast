# OffBlast 0.8.1-RC1 Test Plan

## Release Candidate Testing

This document outlines testing needed before tagging official 0.8.1 release.

## Breaking Changes - IMPORTANT

**Database format changed**: `ownerTag` field added to LaunchTarget structure.

**Required before testing:**
```bash
rm ~/.offblast/launchtargets.bin
rm ~/.offblast/descriptions.bin
```

## Critical Tests (Could Break Core Functionality)

### Multi-User Steam Account Switching
- [ ] Set up multiple Steam accounts with "Remember Password" on test machine
- [ ] Configure different `steam_account_name` for each OffBlast user
- [ ] Test switching between users - verify Steam restarts and logs into correct account
- [ ] Verify each user sees only their own Steam games (not other users' games)
- [ ] Test with user who has no `steam_account_name` - should skip Steam gracefully
- [ ] Verify ROMs/retro games visible to all users (no ownerTag filtering)
- [ ] Test launching Steam game - verify it launches under correct Steam account
- [ ] Test playtime tracking - verify playtime goes to correct user's Steam account
- [ ] Check loading screen appears with user avatar during Steam import
- [ ] Verify progress counter shows during metadata fetching

### VDF Parser Reliability
- [ ] Test with single Steam account in loginusers.vdf
- [ ] Test with multiple Steam accounts in loginusers.vdf
- [ ] Test parsing different VDF formatting (tabs vs spaces)
- [ ] Verify MostRecent field correctly identified and modified
- [ ] Verify AccountName extracted correctly (no whitespace/garbage)

### Database Owner Tagging
- [ ] After import, check Steam games have ownerTag set (not empty)
- [ ] Verify ROM games have empty ownerTag
- [ ] Test filtering: user A doesn't see user B's Steam games
- [ ] Test search results: only shows current user's Steam games
- [ ] Test "Jump Back In" row: only shows current user's Steam games
- [ ] Test Essential Steam row: only shows current user's games (or hidden if no steam_account_name)

## Important Tests (New Features)

- [ ] Verify achievement count appears: " | X/Y Achievements (Z%)"

- [ ] Test with raw ROM file (no .zip) - verify hashing works
- [ ] Play a game, unlock achievement, exit game
- [ ] Verify achievement count updates automatically (X+1/Y)
- [ ] Test with game not in RA cache - should skip silently
- [ ] Test with user who has no RA credentials - should skip gracefully
- [ ] Verify database caching: second view instant, no re-hashing

### Achievement Browser (RA)

- [ ] Close and reopen - verify badges load from cache (instant)
- [ ] Press B to close - verify browser closes cleanly

## Known Issues to Document

- [ ] VDF parser may need refinement for edge cases
- [ ] Steam restart takes 10-15 seconds (unavoidable)

- [ ] Achievement browser visual improvements needed (documented in ROADMAP)

## Sign-off Criteria for Official 0.8.1 Release

- [ ] No crashes or segfaults during normal usage
- [ ] Multi-user Steam switching works reliably
- [ ] RetroAchievements hash verification works with common ROM formats
- [ ] No data loss or corruption
- [ ] Performance acceptable with large libraries
- [ ] Documentation complete (README, CHANGELOG, ROADMAP)
- [ ] AppImage builds successfully with new dependencies (minizip, rcheevos)

## Notes

- Test on clean database (wipe launchtargets.bin and descriptions.bin)
- Test with multiple family members if possible
- Test on different Linux distros if available (Pop!_OS, Ubuntu, Arch, etc.)
- Monitor console output for errors/warnings
- Report any issues to GitHub before official release
